REST: Representational State Transfer
월드 와이드웹(www) 와 같은 하이퍼미디어 시스템을 위한 소프트웨어 아키텍쳐
REST 서버는 클라이언트로 하여금 HTTP 포로트콜을 사용해 서버의 정보에 접근 및 변경을 가능하게함
정보는 text, xml, json 등 형식으로 제공

HTTP 요청 메서드
GET : 서버 자원을 가져오려고 할 때 사용
POST : 서버에 자원을 새로 등록하고자 할때 사용(또는 뭘 써야할 지 애매할 때, 로그인 동작등)
PUT : 서버의 자원을 요청에 들어있는 자원으로 치환하고자 할때 사용
PATCH : 서버 자원의 일부만 수정하고자 할때 사용
DELETE : 서버의 자원을 삭제하고자 할때 사용

서버 주소 구조

HTTP 메서드           주소                     역할
GET             /                   restFront.html 파일제공
GET             /about                about.html 파일 제공
GET             /uwers                사용자 목록 제공
GET             기타                    기타 정적 파일 제공
POST            /user                   사용자 등록
PUT             /user/사용자id          해당 id의 사용자 수정
DELETE          /users/사용자id         해당 id의 사용자 제거
